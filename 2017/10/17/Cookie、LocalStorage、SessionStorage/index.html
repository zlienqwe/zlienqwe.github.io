Cookie、LocalStorage、SessionStorage

发表于 2017-10-17

asadasdasda

cookie

小，只有4K，主要用作有保存登录信息，比如登陆的时候做的记住密码功能,同域可共享一套cookie

原生写法：document.cookie = 'name=xiaojiecong;expires=';

原生API不好用 最好自己封装三个方法

setCookie的方法传入key，value，过期expires，

getCookie方法传入key，

clearCookie方法传入key(在方法里面调setCookie,过期时间设置成-1)

注意：设置cookie时需要使用encodeURIComponent()编码，取值的时候需要使用decodeURIComponent()解码才能读取完整的cookie值

LocalStorage

以前有一个叫userData的东西用于本地存储，现在LocalStorage被大部分浏览器支持了

存储时候会自动类型转换成string，所以存储器其他格式要手动转化

localStorage还可以实现同域页面之间通讯

SessionStorage

session 将一部分数据保存在当前会话中，当前页面关闭自动消失

用法和localstorage一样

不同点

生命周期

  • cookie 一般由服务器生成，可以设置失效时间

  • localstorage 需要手动清除，否则一直存在

  • session 当前会话有效，关掉页面或者浏览器失效

大小

  • cookie 一般4K左右

  • localstorage session 一般5M，所以存储文本信息一般不会用完

与服务器端通信

cookie因为HTTP是完全无状态的协议，每次都相当于全新的一次通讯，每次都会携带在HTTP头中，如果使用cookie保存过多数据会带来性能问题

localstorage session 仅在客户端（即浏览器）中保存，不参与和服务器的通信

安全性

cookie中最好不要放置任何明文的东西

两个storage的数据提交后在服务端一定要校验，因为数据是随时可以手动更改的

使用它们的时候，需要时刻注意是否有代码存在 XSS 注入的风险。
